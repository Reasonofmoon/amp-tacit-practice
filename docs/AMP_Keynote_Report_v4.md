## 1. 강연 개요

### 1.1 강연 제목

**"암묵지에서 AI 앱까지 — 20년 교육 경험을 8개의 도구로 증폭시킨 여정"**

### 1.2 핵심 메시지

본 강연을 통해 참석하신 학원 원장님들께 다음 메시지를 전달합니다.

1. 선생님들이 이미 가진 20년 암묵지는 **AI가 대체할 수 없는 금광**입니다.
2. 그 금광을 **SECI 모델 + AI 프롬프팅**으로 끌어내면, 한 사람이 읽기 진단 도구부터 영어도서관 플랫폼까지 만들 수 있습니다.
3. 증거는 지금부터 보여드립니다.

### 1.3 대상 청중

- 학원 운영자 및 원장 (개발 지식이 없는 비개발자)
- 교육 담당자, 에듀테크에 관심 있는 교육 종사자

## 3. 파트별 상세 내용

### Part 1. 나의 이야기 — 보이지 않는 금광 (15분)

강연자 본인의 실제 경험을 스토리텔링으로 전달합니다. "자전거 타는 법을 말로 설명해보세요" 질문으로 암묵지의 개념을 체감시킵니다.

- 학원 운영자의 일상적 고충 공감 — 반복 업무에 소진되는 시간
- 우리가 가진 지식의 80–90%는 암묵지 — 말로 설명할 수 없지만 몸이 아는 것
- AI 전환점: "이 암묵지를 끌어내서 AI와 결합하면 어떻게 될까?"

> **전환 멘트**: "제 20년 교육 경험은 사라지지 않았습니다. AI와 결합하여 8개의 앱이 되었습니다. 그 비밀은 암묵지에 있습니다."

---

### Part 2. 암묵지 이야기 — SECI 모델 + 5단계 (25분)

암묵지의 정의와 중요성을 설명하고, 노나카의 SECI 모델(사회화→표출화→연결화→내면화)을 통해 암묵지를 형식지로 전환하는 프레임워크를 소개합니다.

- **암묵지 정의**: 경험으로 체화된 지식. 말로 표현하기 어렵지만 전문성의 핵심
- **SECI 모델**: 노나카 이쿠지로의 지식 창조 이론을 교육 현장에 적용
- **5단계 프레임워크**: 경험 회상 → 패턴 인식 → 구조화 → AI 결합 → 앱으로 구현

> **전환 멘트**: "이 5단계를 거쳐 탄생한 앱들을 쉬운 것부터 어려운 것 순서로 보여드리겠습니다. 비개발자인 제가 처음 만든 것부터 시작합니다."

---

### Part 4. 암묵지 워크샵 + Q&A (30분)

일방적 강연으로 끝나지 않고, 청중이 직접 자신의 암묵지를 꺼내보는 상호작용 시간을 갖습니다. **이 세션이 강연의 핵심**입니다.

#### 암묵지 끌어내기 워크샵 (15분)

**Step 1. 개인 암묵지 발굴 (5분)**

청중 각자가 종이에 다음 질문에 답합니다.

1. **"나만 아는 것"** — 학원 운영에서 내가 가장 잘하는 것, 남들은 모르지만 항상 하고 있는 것은?
2. **"반복되는 패턴"** — 매번 비슷한 방식으로 해결하지만, 말로 설명하라면 막막해지는 것은?
3. **"가장 힘든 것"** — 가장 시간이 많이 드는 반복 업무, 자동화되면 좋겠다고 느끼는 것은?

**Step 2. 짝 공유 — 사회화(Socialization) 훈련 (5분)**

옆 자리의 2–3명과 짝을 지어 서로의 암묵지를 공유합니다. SECI 모델의 첫 번째 단계인 "사회화" — 암묵지를 말로 꺼내는 훈련입니다.

- "저는 학부모 상담할 때 항상 ___를 먼저 합니다" → 이것이 암묵지
- "성적이 떨어지는 학생을 보면 ___를 먼저 확인합니다" → 이것이 암묵지
- 서로 들으며 "아, 나도 그런 것이 있었구나" 하는 발견의 순간을 경험

**Step 3. 표출화(Externalization) — AI 프롬프트로 전환 (5분)**

방금 적은 암묵지를 실제로 AI 프롬프트로 바꾸어 봄으로써, SECI 모델의 두 번째 단계를 체험합니다.

- 강연자가 라이브로 청중 1명의 암묵지를 ChatGPT/Claude 프롬프트로 전환하는 과정을 시연
- 예시: "학부모 상담 시 항상 성적보다 태도를 먼저 언급한다" → 이것을 AI 피드백 시스템의 프롬프트로 변환
- 청중이 "아, 내 암묵지도 이렇게 할 수 있겠다" 하는 체감을 유도

#### Q&A + 라이브 앙코르 (15분)

- 예상 질문 10개 사전 준비
- 청중 요청 시 앱 추가 시연 가능

## 5. v3.1 → v4.0 변경 요약

| 항목 | v3.1 (이전) | v4.0 (현재) |
|---|---|---|
| **앱 수** | 7개 | **8개** |
| **시연 1–2** | English Newsletter + Book Parser (GAS) | **ReadMaster + Pet Trip** (Vercel 원클릭) |
| **시연 3–4** | Academy OS + Moon-Tab Flow | **SmartStart + Edu Ontology** (AI API 연동) |
| **시연 5–6** | Dark Moon Subtitler + MoonLang | **Knot + BlueL** (풀스택 웹앱) |
| **시연 7** | Sabo Philosophy (피날레) | **Librainy + MoonLang** (종합 플랫폼 + 수익화) |
| **메인 네비게이터** | 슬라이드 전환 | **쇼케이스 앱 Monopoly Path** 화면 |
| **난이도 축** | GAS → Chrome → Python → React | **Vercel 배포 → AI API → 풀스택 → 플랫폼/수익화** |

> **설계 의도**: 기존 v3.1이 **기술 스택**(GAS → Python → React) 기준으로 난이도를 나눴다면, v4.0은 **교육적 임팩트**(진단 → 동기부여 → 지식관리 → 생태계) 기준으로 재배열하여, 비개발자 청중이 "이것에서 저것으로 발전했구나"라는 **성장 서사**를 자연스럽게 체감하도록 설계했습니다.
