> *(storyText: "단순한 정보 제공을 넘어, 유용한 AI 툴 정보와 레퍼럴 코드 사이의 중개를 통해 실제 비즈니스 수익 창출이 가능한지 검증하는 실험 모델")*
> 
> "마지막 앱입니다. 이건 좀 다릅니다. 교육 앱이 아니에요. **수익화 실험 모델**입니다."
> 
> "저는 이 앱들을 만드는 과정에서 수백 가지 AI 도구를 사용했습니다. 그러다 보니 **어떤 도구가 진짜 쓸모있고, 어떤 게 허접한지** 아는 안목이 생겼어요. 이것도 암묵지죠."
> 
> "그래서 생각했습니다. '이 안목 자체를 비즈니스로 만들 수 없을까?'"

#### Phase B — 라이브 앱 시연

> **대사:**
> 
> **[시연 동선]**
> 1. tools.moonlang.com 메인 — "AI 도구를 카테고리별로 정리한 디렉토리입니다"
> 2. 카테고리 탐색 — "Writing, Image, Video, Coding… 이렇게 분류되어 있고"
> 3. 수익 모델 설명 — "각 도구에 레퍼럴 링크가 들어가 있습니다. 누군가 이 링크를 통해 가입하면 수수료가 발생합니다"
> 
> "**핵심 메시지는 이겁니다:** 여러분의 전문 지식 — 어떤 교재가 좋고, 어떤 학습법이 효과적인지 아는 그 안목 — 이것 자체가 **콘텐츠이자 비즈니스**가 될 수 있습니다."
> 
> "제가 AI 도구 큐레이션으로 했듯이, 원장님이 '초등영어 교재 추천 사이트'를 만들면? 그 안에 제휴 링크를 넣으면? **암묵지가 월 수입**이 됩니다."

**[액션]** 체험 완료 → 보드 복귀 → 전체 노드 완료 상태 → 🎊 컨페티 대폭발

> **클로징 멘트:**

## 📌 앱 개선 제안 (Slide-Enhancement Proposals)

현재 쇼케이스 앱을 **키노트 프레젠테이션 도구**로 더 효과적으로 사용하기 위한 개선 제안입니다.

---

### 제안 1: Phase A에 "강연자 노트(Speaker Notes)" 패널 추가

| 항목 | 내용 |
|---|---|
| **문제** | 현재 Phase A는 storyText 한 줄만 표시됨. 강연자가 이 단계에서 할 말을 따로 외워야 함 |
| **제안** | `PresentationShell.jsx`에 **프레젠터 뷰** 모드 추가. 키보드 `P` 키를 누르면 하단에 반투명 강연자 노트가 표시됨 |
| **데이터** | `showcaseActivities.js`에 `speakerNotes` 필드 추가 |
| **우선도** | ⭐⭐⭐⭐⭐ (가장 실전적) |

### 제안 2: 쇼케이스 노드에 "난이도 뱃지" 시각화

| 항목 | 내용 |
|---|---|
| **문제** | 현재 모노폴리 보드의 각 노드가 난이도를 색깔로만 구분. 청중이 "쉬운 것→어려운 것" 흐름을 인지하기 어려움 |
| **제안** | 각 노드 상단에 **⭐ 난이도 별표** 뱃지 표시 (1~4개). 레벨별 구간 구분선(점선) 추가 |
| **데이터** | `showcaseActivities.js`에 `difficulty: 1~4` 필드 추가 |
| **우선도** | ⭐⭐⭐⭐ |

### 제안 3: 앱 간 전환 시 "진행률 프로그레스바" 추가

| 항목 | 내용 |
|---|---|
| **문제** | 8개 앱 중 지금 몇 번째인지 청중이 파악하기 어려움 |
| **제안** | 화면 상단에 **"2/8 완료"** 같은 프로그레스바를 고정. 완료된 노드에는 체크마크 |
| **구현 위치** | `Layout.jsx` 또는 `App.jsx`의 쇼케이스 여정 전용 헤더 |
| **우선도** | ⭐⭐⭐⭐ |

### 제안 4: Phase A에 "암묵지 → 형식지" 변환 애니메이션 강화

| 항목 | 내용 |
|---|---|
| **문제** | 현재 Phase A → B 전환이 로딩 스피너만 2.5초. 시각적으로 약함 |
| **제안** | 전환 단계에서 **"암묵지(직관) → 프롬프트(언어) → 코드(구현) → 앱(결과)"** 4단계가 순차적으로 타이핑되는 애니메이션 추가. SECI 모델의 표출화 과정을 눈으로 보여줌 |
| **구현 위치** | `PresentationShell.jsx`의 `phase === 'transition'` 섹션 |
| **우선도** | ⭐⭐⭐⭐⭐ (강연의 핵심 메시지와 직결) |

### 제안 5: Level 구간별 "브리지 슬라이드" 추가

| 항목 | 내용 |
|---|---|
| **문제** | Level 1→2→3→4로 넘어갈 때 구두로만 전환 멘트를 함. 시각적 구분이 없음 |
| **제안** | 레벨 전환 시 전체화면 **브리지 슬라이드** 자동 표시. "🟢 Level 1 Complete → 🔵 Level 2: AI를 연결합니다" 같은 대형 텍스트 + 아이콘 |
| **구현** | 쇼케이스 보드에서 특정 노드 완료 시 자동 트리거되는 인터스티셜 화면 |
| **우선도** | ⭐⭐⭐ |

### 제안 6: 각 노드에 "핵심 기술 태그" 표시

| 항목 | 내용 |
|---|---|
| **문제** | 현재 노드 카드에 기술 스택 정보가 없음. 청중이 각 앱의 기술 복잡도를 시각적으로 비교 불가 |
| **제안** | 각 `ActivityCard`에 작은 태그 칩을 추가: `React`, `AI API`, `Next.js`, `Supabase` 등 |
| **데이터** | `showcaseActivities.js`에 `techTags: ['React', 'Vercel']` 필드 추가 |
| **우선도** | ⭐⭐⭐ |

### 제안 7: "청중 참여 QR 코드" 인터스티셜
